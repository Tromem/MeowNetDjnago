Для работы с проектом убедитесь, что у вас установлен Python 3.10+ и pip. Сначала клонируйте проект командой git clone <URL_репозитория> и перейдите в папку проекта. Создайте виртуальное окружение: на Windows используйте python -m venv venv и активируйте его командой venv\Scripts\activate, на macOS/Linux — python3 -m venv venv и source venv/bin/activate. После активации виртуального окружения в консоли будет отображаться (venv).

Все необходимые библиотеки проекта перечислены в файле requirements.txt. Чтобы их установить, выполните команду pip install -r requirements.txt. Это гарантирует, что будут установлены все версии библиотек, которые использовались при разработке проекта.

Если проект использует переменные окружения, создайте файл .env в корне проекта и добавьте туда необходимые параметры, например: DJANGO_SECRET_KEY, DEBUG=True, DATABASE_URL=sqlite:///db.sqlite3. Не храните реальные секретные ключи в публичных репозиториях.

Примените миграции командой python manage.py migrate для создания всех таблиц базы данных. Для доступа к админ-панели создайте суперпользователя с помощью python manage.py createsuperuser и следуйте инструкциям в консоли.

Запустите сервер разработки командой python manage.py runserver. После этого проект будет доступен по адресу http://127.0.0.1:8000/. При необходимости можно собрать статические файлы командой python manage.py collectstatic, а для запуска тестов использовать python manage.py test.

Полезные команды Django: python manage.py makemigrations для создания миграций на основе изменений моделей, python manage.py migrate для их применения, python manage.py shell для интерактивной работы с проектом, python manage.py runserver для запуска сервера разработки.

Все зависимости и версии библиотек указаны в requirements.txt, поэтому для корректной работы проекта достаточно устанавливать их именно из этого файла.
